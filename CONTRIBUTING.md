# Contributing

Thanks for your interest in improving REscript.

## Before You Start

- keep secrets out of commits
- do not commit runtime data from `data/`
- prefer focused changes over broad refactors
- validate with `npm run build` before opening a pull request

## Pull Requests

Please include:

- what changed
- why it changed
- any deployment or environment impact
- screenshots for public UI changes when relevant

## Scope

Maintainers prioritize:

- security fixes
- deployability
- billing and delivery reliability
- public-site quality

Large product-direction changes may be deferred even if technically sound.
